# Contributing

## Branch naming
`type/short-description`, e.g. `feat/login-page`, `fix/header-overflow`

## PR title
Same `type` prefix, e.g. `feat: add login page`

## Types
- `feat` – new feature
- `fix` – bug fix
- `chore` – maintenance, no user-facing change
- `refactor` – code change with no behavior change
- `docs` – documentation only
- `test` – adding or fixing tests
